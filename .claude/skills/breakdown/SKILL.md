# /breakdown

1. Read `docs/workflow.md`.
2. Capture the problem, in-scope items, out-of-scope items, and key terms.
3. Start from `templates/feature-init.md` or `templates/revamp-init.md`.
4. Prefer `npm run new:spec -- <feature-name>` if the repo owner wants a new starter-aligned file scaffold.
5. Write the draft to `research/` or `specs/`, depending on how mature the request is.
6. Keep the output short, decision-oriented, and ready for `/design`.
