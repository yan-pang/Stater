# PRD Checklist

## G0 Requirement Readiness

- Goal is explicit
- Scope boundaries are explicit
- Terms are defined
- Existing state is described
- Open questions are visible

## G1 Design Completeness

- Main flow is described
- Fields and rules are described
- Edge cases are described
- Acceptance is testable
- Dependencies are visible

## G2 Handoff Readiness

- External wording is audience-appropriate
- Requirement IDs are stable
- Risks and assumptions are visible
- Test strategy maps back to requirements
- Stale placeholder sections are removed
