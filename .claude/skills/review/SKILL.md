# /review

1. Review findings first, not summaries.
2. Check for stale terminology, dead links, spec gaps, and missing acceptance coverage.
3. If reviewing code, point to concrete files and behaviors.
4. If reviewing workflow docs, suggest the smallest change that removes duplication.
5. For starter-level changes, confirm `README.md`, `docs/workflow.md`, and `.github/workflows/ci.yml` still agree.
